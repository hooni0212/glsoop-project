# 웹페이지 개선 제안 (메인 홈 기준)

## 관찰 사항과 개선 방향

1. **SEO/미리보기 메타 태그 부재**  
   메인 페이지 `<head>`에는 문자 인코딩, 뷰포트, 제목만 정의되어 있고 검색/공유에 필요한 `description`, Open Graph, Twitter 카드 메타가 없다.【F:public/index.html†L3-L35】  
   → 주요 문구를 담은 `meta name="description"`, OG `title/description/image`, `canonical` 링크를 추가해 검색 결과와 소셜 공유 시 클릭률을 높이세요.

2. **불필요한 폰트·테마 CSS 동시 로드**  
   Google 폰트 3종과 계절 테마 CSS 4종을 모든 방문자에게 로드하고 있어 초기 페이로드가 크고, 실제로 한 계절 테마만 필요할 때 낭비가 발생한다.【F:public/index.html†L14-L35】  
   → 필수 기본 폰트 1~2종만 우선 적용하고, 테마 CSS는 사용자 선택/계절에 따라 지연 로드하거나 한 번에 하나만 적용하도록 분기하세요.

3. **키보드 포커스 가시성 부족**  
   내비게이션과 CTA는 호버 스타일만 정의돼 있어 키보드 탐색 시 현재 초점 위치를 식별하기 어렵다.【F:public/css/main.css†L87-L136】【F:public/index.html†L150-L178】  
   → `.nav-link`, `.feed-tab`, `.hero-cta-btn` 등에 `:focus-visible` 테두리·섀도우를 추가해 접근성을 개선하세요.

4. **히어로 섹션 상단 메시지와 CTA 간 연계 강화 필요**  
   히어로 설명이 정적인 문장만 제공하고 CTA가 바로 글쓰기로 연결되어, 신규 사용자가 서비스 가치를 이해하기 전에 이탈할 수 있다.【F:public/index.html†L122-L141】  
   → 핵심 가치 제안과 최근 인기 글/태그 링크, 짧은 온보딩(예: "1분 만에 가입, 익명 발행 가능")을 추가해 CTA 설득력을 높여보세요.

5. **피드 로딩/빈 상태 안내의 시각적 피드백 한계**  
   피드가 로딩되거나 글이 없을 때 단순 문구만 표시되어, 느린 네트워크 환경에서 진행 상태를 체감하기 어렵다.【F:public/index.html†L182-L188】【F:public/js/index.js†L224-L319】  
   → 스켈레톤 카드나 스피너, 재시도 버튼을 제공해 사용자가 기다리는 동안 명확한 피드백을 받도록 개선하세요.

## 우선순위 제안
- **1순위:** 메타 태그 추가 및 폰트/테마 로딩 최적화로 SEO와 초기 속도를 개선.
- **2순위:** 키보드 포커스 스타일과 로딩/빈 상태 UI로 접근성과 체감 품질 강화.
- **3순위:** 히어로 카피 및 CTA 주변에 온보딩/추천 콘텐츠를 배치해 전환율 상승 유도.
